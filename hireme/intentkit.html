IntentKit

http://intentkit.github.io

Role: UX Architecture, Visual Design and Iconography, Code

IntentKit is an iOS library to make inter-app linking simpler. It's one of the most popular open-source iOS libraries, with over 1,000 stars on GitHub, and has been written about in prestigious iOS developer journals such as NSHipster.

Android and Windows Phone both allow users to set default applications for common tasks like mapping and sending email. iOS doesn't provide a way to do this, despite many users (for example) preferring Google Maps to Apple Maps. A handful of third-party apps build their own custom solutions, but because a great deal of engineering work is required the experience is usually clunky and thrown-together for the sake of time.

IntentKit attempts to solve that problem. It provides a widget developers can drop into their apps to let power users pick and set preferences for third-party apps, while staying out of the way for users who don't care. My design goal was to create a workflow that felt like the one Apple themselves would design.

There are two ways IntentKit can be used:

  • Apps that have a settings screen can include an IntentKit settings pane that lets users set their preferences, defaulting to Apple's standard apps if the user doesn't specify anything.

  • Apps without an explicit settings screen can show an "Open In" modal sheet, inspired by Apple's UIActivityViewController sheet, prompting users to pick from all appropriate installed applications and letting them save their preference for the future. If a user doesn't have any third-party applications installed, the sheet isn't shown at all.

It even works for activities that can be done in-app, such as letting users specify whether they'd prefer to send email by using an in-app share sheet or going to a third-party app like Mailbox or the native Gmail app.
